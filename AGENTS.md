# OpenCode 在 Obsidian Vault 中的使用指南

## 工作目录
本 vault 位于 D:\第二大脑

## 核心能力
- 使用 `/obsidian-inbox` 整理收件箱笔记
- 使用 `/wechat-writer` 辅助公众号写作
- 使用 `/everything-search` 全局搜索 vault
- 使用 `/excalidraw-draw` 创建图表
- 使用 `/feishu-inbox` 手动处理一条内容（链接/文字/文件）

## 飞书 Bot 自动处理流程

当接收到来自飞书机器人的消息（URL、灵感文字、笔记、文件内容等），请自动执行以下流水线：

### 处理规则
1. **判断内容类型**：链接（公众号/小红书/B站/网页） / 灵感文字 / 文件内容
2. **处理链接**：用 MCP 的 fetch 工具或浏览器获取页面内容，提取正文
3. **分类**：根据内容判断类别，放入对应文件夹
   - 技术/知识类 → 根目录
   - 写作灵感/素材 → `写作/` 或 `写作资料/`
   - 公众号选题 → `公众号/`
   - 纯临时/未分类 → `Inbox/`
4. **生成结构化 Markdown**，包含：
   - YAML frontmatter：`title`、`created`（日期）、`tags`、`source`（来源URL连）、`status: inbox`
   - 内容体：摘要、要点、完整内容、我的想法（留空）
   - Obsidian 内部使用 `[[wikilinks]]` 和 `#tag` 格式
5. **保存文件**：文件名格式为 `YYYYMMDD-{简短英文/拼音标题}.md`，保存到对应分类文件夹
6. **回复确认**：回复飞书消息，告知保存位置（文件夹/文件名）和简要摘要

### 注意
- 只处理非 `/` 开头的普通消息（命令消息走命令逻辑）
- 禁止修改已存在的笔记，只创建新文件
- 不要询问用户确认，直接执行并回复结果

## 知识库结构
- `写作/` - 短篇写作资料（大纲、人设、素材）
- `写作资料/` - 导入的写作素材
- `公众号/` - 公众号文章
- `Inbox/` - 未处理/临时收件箱
- 根目录 - 日常笔记

## Obsidian 格式约定
- 使用 `[[wikilinks]]` 链接 vault 内的笔记
- 使用 `![[embed]]` 嵌入其他笔记内容
- 使用 `> [!note]` 标注重点
- 使用 `#tag` 添加标签
- 在文件开头添加 YAML frontmatter（title、tags、aliases、created date）
